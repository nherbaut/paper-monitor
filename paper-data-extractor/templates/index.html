Metadata

A Data Extraction Model is a reusable extraction model fragment. It defines dimensions, taxa, criteria, scales, and rules that can be composed into a Review Design.

Optional source details

Scales

A scale defines the allowed answers for criteria. Criteria reference scales by id, so define reusable scales such as yes/no, ordinal 0-2, or numeric score here.

Dimensions

A dimension is one classification axis and becomes a form field. Category dimensions contain taxa; dimensions can also contain nested subdimensions for hierarchical models.

Add at least one dimension.

Rules

Rules document how the taxonomy should be applied. They are stored with the model but are not executable validation constraints yet.
